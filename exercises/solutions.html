Part 1: filter() Family

Solution 1 → Exercise 1: filter_out()

Task: Drop rows where island == "Torgersen" and body_mass_g > 4000 are both true. Rows with NA should be kept.

penguins_small <- tibble(
  species     = rep("Adelie", 5),
  island      = c("Torgersen", "Biscoe", "Dream", "Torgersen", "Torgersen"),
  body_mass_g = c(3750, 3800, 3300, 4500, NA_real_)
)
# filter_out() drops rows where ALL listed conditions are TRUE.
# Rows where any condition is NA are kept — no extra is.na() calls needed.
penguins_small |>
  filter_out(island == "Torgersen", body_mass_g > 4000)
# A tibble: 4 × 3
  species island    body_mass_g
  <chr>   <chr>           <dbl>
1 Adelie  Torgersen        3750
2 Adelie  Biscoe           3800
3 Adelie  Dream            3300
4 Adelie  Torgersen          NA

Why filter_out() is better:

  • Intent is explicit: you’re dropping, not keeping
  • NA rows are preserved automatically
  • No need to negate (!) or manually guard against NAs

Solution 3 → Exercise 3: when_any() + when_all() (complex OR)

Task: Keep rows where: - Group A: species == "Adelie" AND island == "Torgersen" AND body_mass_g > 3700, or - Group B: species == "Gentoo" AND body_mass_g > 5000

penguins_filters <- tibble(
  species     = c("Adelie", "Adelie", "Adelie", "Adelie",
                  "Gentoo", "Gentoo", "Gentoo", "Gentoo"),
  island      = c("Torgersen", "Torgersen", "Dream", "Biscoe",
                  "Biscoe",    "Dream",     "Biscoe", "Dream"),
  body_mass_g = c(3750, 3800, 3250, NA_real_,
                  4500, 5700, 4450, 5700)
)
# when_all() bundles AND conditions into one unit.
# when_any() then applies OR across those bundles.
penguins_filters |>
  filter(
    when_any(
      when_all(species == "Adelie", island == "Torgersen", body_mass_g > 3700),
      when_all(species == "Gentoo", body_mass_g > 5000)
    )
  )
# A tibble: 4 × 3
  species island    body_mass_g
  <chr>   <chr>           <dbl>
1 Adelie  Torgersen        3750
2 Adelie  Torgersen        3800
3 Gentoo  Dream            5700
4 Gentoo  Dream            5700

Why when_any() + when_all() is better:

  • Groups AND conditions together visually instead of relying on () nesting
  • The two groups are clearly separated — easy to add a third group
  • Matches how you’d describe the logic in plain English: “any of these groups”

Solution 4 → Exercise 4: when_all() (AND)

Task: Keep rows where species == "Gentoo" and body_mass_g > 5000.

penguins_when_all <- tibble(
  species     = c("Adelie", "Adelie", "Adelie", "Adelie",
                  "Gentoo", "Gentoo", "Gentoo", "Gentoo"),
  island      = c("Torgersen", "Dream", "Biscoe", "Torgersen",
                  "Biscoe",    "Dream", "Biscoe", "Dream"),
  body_mass_g = c(3750, 3800, 3250, NA_real_,
                  4500, 5700, 4450, 5700)
)
# when_all() makes AND logic explicit.
penguins_when_all |>
  filter(when_all(species == "Gentoo", body_mass_g > 5000))
# A tibble: 2 × 3
  species island body_mass_g
  <chr>   <chr>        <dbl>
1 Gentoo  Dream         5700
2 Gentoo  Dream         5700

Why when_all() is better:

  • Symmetrical with when_any() — one consistent set of verbs for boolean logic
  • Easy to nest inside when_any() when your logic grows more complex
  • For simple AND, commas in filter() still work fine too

Rule of thumb: For simple & conditions, plain commas in filter() are fine. Reach for when_all() when you need to nest it inside when_any().

Part 2: Recoding Values

Solution 5 → Exercise 5: recode_values() inline

Task: Add an island_location column using the mapping Biscoe → Southwest, Dream → Northwest, Torgersen → Northwest.

penguins_islands <- tibble(
  species     = c("Adelie", "Adelie", "Adelie", "Gentoo"),
  island      = c("Biscoe", "Dream", "Torgersen", "Biscoe"),
  body_mass_g = c(3750, 3800, 4750, 5000)
)
# recode_values() uses formula syntax: "old_value" ~ "new_value"
# No repetitive == checks needed.
penguins_islands |>
  mutate(
    island_location = island |>
      recode_values(
        "Biscoe"    ~ "Southwest",
        "Dream"     ~ "Northwest",
        "Torgersen" ~ "Northwest"
      )
  )
# A tibble: 4 × 4
  species island    body_mass_g island_location
  <chr>   <chr>           <dbl> <chr>          
1 Adelie  Biscoe           3750 Southwest      
2 Adelie  Dream            3800 Northwest      
3 Adelie  Torgersen        4750 Northwest      
4 Gentoo  Biscoe           5000 Southwest

Why recode_values() is better:

  • No repeated island == on every line
  • Formula syntax ("old" ~ "new") clearly states input → output
  • Unmatched values are NA by default; use unmatched = "error" to catch surprises

Solution 6 → Exercise 6: recode_values() with a lookup table

Task: Same island → location mapping, but sourced from the island_lookup table.

island_lookup <- tibble(
  from = c("Biscoe", "Dream", "Torgersen"),
  to   = c("Southwest", "Northwest", "Northwest")
)

island_lookup
# A tibble: 3 × 2
  from      to       
  <chr>     <chr>    
1 Biscoe    Southwest
2 Dream     Northwest
3 Torgersen Northwest
# Pass the lookup table's columns directly to from = / to =
penguins_islands |>
  mutate(
    island_location = island |>
      recode_values(
        from = island_lookup$from,
        to   = island_lookup$to
      )
  )
# A tibble: 4 × 4
  species island    body_mass_g island_location
  <chr>   <chr>           <dbl> <chr>          
1 Adelie  Biscoe           3750 Southwest      
2 Adelie  Dream            3800 Northwest      
3 Adelie  Torgersen        4750 Northwest      
4 Gentoo  Biscoe           5000 Southwest

Why recode_values() with a lookup table is better:

  • The mapping lives in a data frame — easy to load from a CSV or database
  • Adding or changing a mapping means editing the table, not the pipeline
  • Works seamlessly with from = / to = vectors of any length

Solution 7 → Exercise 7: replace_values()

Task: In island_location, replace "Northwest" with "Northern Palmer Archipelago". Leave other values unchanged.

penguins_locations <- tibble(
  species         = c("Adelie", "Adelie", "Adelie", "Gentoo"),
  island          = c("Biscoe", "Dream", "Torgersen", "Biscoe"),
  body_mass_g     = c(3750, 3800, 4750, 5000),
  island_location = c("Southwest", "Northwest", "Northwest", "Southwest")
)
# replace_values() replaces exact matches in place.
# Unmatched values are left as-is — no .default needed.
penguins_locations |>
  mutate(
    island_location = island_location |>
      replace_values("Northwest" ~ "Northern Palmer Archipelago")
  )
# A tibble: 4 × 4
  species island    body_mass_g island_location            
  <chr>   <chr>           <dbl> <chr>                      
1 Adelie  Biscoe           3750 Southwest                  
2 Adelie  Dream            3800 Northern Palmer Archipelago
3 Adelie  Torgersen        4750 Northern Palmer Archipelago
4 Gentoo  Biscoe           5000 Southwest

Why replace_values() is better:

  • No .default = island_location boilerplate
  • Intent is clear: you’re replacing a value, not rebuilding the column
  • Use multiple formulas to replace several values in one call

Solution 8 → Exercise 8: replace_when()

Task: Cap body_mass_g at 5000 g. Values above 5000 become 5000; everything else stays the same.

penguins_large <- tibble(
  species     = c("Adelie", "Adelie", "Gentoo", "Gentoo"),
  island      = c("Biscoe", "Dream",  "Biscoe", "Biscoe"),
  body_mass_g = c(3750, 3800, 5200, 5400)
)
# replace_when() takes: the column to modify, then condition ~ replacement pairs.
# Rows that don't match any condition keep their original value automatically.
penguins_large |>
  mutate(
    body_mass_g = replace_when(
      body_mass_g,
      body_mass_g > 5000 ~ 5000
    )
  )
# A tibble: 4 × 3
  species island body_mass_g
  <chr>   <chr>        <dbl>
1 Adelie  Biscoe        3750
2 Adelie  Dream         3800
3 Gentoo  Biscoe        5000
4 Gentoo  Biscoe        5000

Why replace_when() is better:

  • No .default = body_mass_g boilerplate
  • Clear that you’re modifying an existing column, not building a new one
  • Use multiple condition ~ value pairs to handle several replacements at once

Quick Reference

Exercise Task Old function New function
1 Drop rows matching all conditions filter(!(cond1 & cond2)) with NA guards filter_out(cond1, cond2)
2 Keep rows matching any condition filter(a \| b) filter(when_any(a, b))
3 Complex grouped OR logic filter((a & b & c) \| (d & e)) filter(when_any(when_all(a, b, c), when_all(d, e)))
4 Keep rows matching all conditions filter(a & b) filter(when_all(a, b))
5 Recode values (inline) mutate(case_when(x == "a" ~ "A", ...)) mutate(recode_values("a" ~ "A", ...))
6 Recode values (lookup table) Manual case_when() from vectors recode_values(from = ..., to = ...)
7 Replace exact values in column case_when(x == "a" ~ "A", .default = x) replace_values("a" ~ "A")
8 Replace values by condition case_when(cond ~ val, .default = x) replace_when(x, cond ~ val)
